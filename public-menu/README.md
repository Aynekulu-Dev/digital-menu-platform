# Digital Menu Platform — Public Menu (standalone)

The diner-facing `/menu/:slug` experience, decoupled from the React admin
dashboard exactly as the SDD specifies (Section 3.2.2: "Vanilla JS /
Lightweight Static HTML... keeps the public-facing QR menu ultra-lightweight").

No framework, no bundler, no `npm install`. Three files:

- `index.html` — shell
- `app.js` — fetches `GET /api/v1/public/menu/{slug}/` and renders it
- `styles.css` — matches the admin dashboard's espresso/paper/turmeric palette
- `app-config.js` — the one file you edit per environment (`API_BASE_URL`)

## Local preview

```bash
cd public-menu
python -m http.server 8080
# then open http://127.0.0.1:8080/menu/<your-test-slug>
```

Point `app-config.js` at your local backend (`http://127.0.0.1:8000`) or the
deployed Render API before testing.

## Deploying to Render

This folder has its own `render.yaml` — deploy it as a **separate Render
Static Site** from the backend/frontend services (Render Blueprints support
multiple `render.yaml` files across a monorepo, or just point a new Static
Site at this subfolder manually):

1. New → Static Site, root directory `public-menu/`.
2. Build command: none needed (`echo` placeholder is fine).
3. Publish directory: `public-menu` (or `.` if root dir is already set to it).
4. Add the rewrite rule from `render.yaml` (`/menu/*` → `/index.html`) in the
   Render dashboard's Redirects/Rewrites tab if it isn't picked up
   automatically from the blueprint.
5. Edit `app-config.js` to point `API_BASE_URL` at your deployed
   `menu-platform-api` service, then redeploy (or just edit-and-save — no
   build step to rerun).
6. Set `PUBLIC_MENU_BASE_URL` on the **backend** service to this static
   site's URL, so QR codes generated by `GET /api/v1/qr/...` link here.

## Why a separate site instead of the React app's `/menu/:slug` route?

Both still work — `PublicMenu.jsx` in the React app wasn't removed. This
gives you the option described in the spec: a diner never downloads the
~150KB+ React/Tailwind bundle just to read a menu on a weak restaurant wifi
connection. Point your QR codes at whichever origin you prefer.
